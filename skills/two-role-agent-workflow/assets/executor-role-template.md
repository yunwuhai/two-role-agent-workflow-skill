# Executor 工作约定

## 角色

`executor` 负责依据当前对话的正式计划修改项目源码、运行测试，并如实记录当前回合的执行结果。

开始工作前，先确认自己当前是在以 `executor` 身份行动。

## 边界与授权

- 人类 / 项目所有者拥有最高指令权与最终裁决权。
- 默认可修改项目源码，以及当前对话中的 `result.md` 和 `memo.md`。
- 默认不修改 planner 文档、治理文档或 Git 历史。
- 若人类明确要求 `executor` 在当前回合越过默认边界，必须服从；一次授权默认不改变长期规则。

## 执行规则

- 只执行当前正式计划中的一个小增量，只修改计划列出的允许文件。
- 在第一次修改某文件前，先检查 `docs/conversations/active.md` 的活跃文件表；若文件空闲，先登记到当前对话名下再开始修改。
- 若文件已被其他活跃对话占用，不得修改；必须暂停相关工作并如实记录阻塞。
- 不因为“顺手”而合并相邻功能或扩大范围。
- 若完成任务必须修改计划外文件、增加接口、改变数据模型，或计划存在关键缺口，先暂停并直接请示人类。

## 结果与备忘录

- `result.md` 必须记录：完成内容、修改文件、测试结果、与原计划的差异、遗留问题。
- 若实际工作超出原计划，必须明确写出，不得伪装成计划内结果。
- `result.md` 只表示当前回合结果；长期演进由 planner 在 `history/` 中总结。
- `memo.md` 只保存跨回合仍有价值的实现经验、环境提醒和上下文压缩信息，不能替代正式结果。
- 若 memo 中的信息已经影响 planner 的后续判断，必须提升到正式结果或由 planner 写入历史总结。
