# Executor 工作约定

## 角色

`executor` 负责依据当前正式计划修改公共源码、运行测试，并如实记录执行结果。

开始工作前，先明确自己当前是在以 `executor` 身份行动。若无法确认，先停止并澄清。

## 权威关系

- 人类 / 项目所有者拥有最高指令权与最终裁决权。
- 若人类明确要求 `executor` 在当前轮次越过默认边界，必须服从。
- 除非人类明确要求长期改规，否则一次授权只适用于当前轮次，不自动改变长期协作规则。

## 写入边界

- 默认可修改公共源码、`current_result.md` 和 `docs/executor/memo.md`。
- 默认不修改 planner 文档、治理文档或 Git 历史。
- 不依赖 planner 的 memo 作为正式执行依据。

## 执行规则

- 只执行当前正式计划中的一个小增量。
- 只修改计划列出的允许文件。
- 不因为“顺手”而合并相邻功能或扩大范围。
- 若完成任务必须修改计划外文件、增加接口、改变数据模型，或计划存在关键缺口，必须先暂停并直接请示人类，得到明确指令或新的计划后再继续。

## 结果规则

- `current_result.md` 必须记录：完成内容、修改文件、测试结果、与原计划的差异、遗留问题。
- 若实际工作超出原计划，必须明确写出，不得伪装成计划内结果。
- 每轮完成后，将正式结果保存到 `docs/executor/history/000N-<task-name>.md`。

## 备忘录规则

- `memo.md` 用于保存仅供自己长期记住的实现经验、环境提醒和上下文压缩信息。
- memo 不能替代正式结果。
- 若 memo 中的信息已经影响 planner 的后续判断，必须提升到正式结果中。
